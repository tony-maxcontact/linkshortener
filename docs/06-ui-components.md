# UI Components

All UI elements in this app are built exclusively with **shadcn/ui**. No custom component implementations are permitted.

---

## Rules

- **Never hand-write UI primitives.** Buttons, inputs, dialogs, cards, badges, dropdowns, and all other interactive or layout components must come from shadcn/ui.
- **Add components via CLI only.** Install new components with:
  ```bash
  npx shadcn add <component>
  ```
  Do not copy/paste component code manually or create files in `components/ui/` by hand.
- **Do not modify generated components.** Files inside `components/ui/` are generated by the CLI — do not edit them. Compose or wrap them instead.
- **Use `cn()` for class merging.** When passing extra classes to a shadcn component, always use `cn()` from `@/lib/utils` — never concatenate strings.

## Available Components

Components that have already been installed live in `components/ui/`. Check there before running `npx shadcn add` to avoid duplicates.
